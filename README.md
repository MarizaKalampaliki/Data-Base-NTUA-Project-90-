# Γενικό Νοσοκομείο Υγειόπολης
## Εξαμηνιαία εργασία του μαθήματος Βάσεων Δεδομένων 2025-2026, ΗΜΜΥ ΕΜΠ, νοσοκομειακό πληροφοριακό σύστημα

Η παρούσα εργασία αφορά την ανάπτυξη ενός συστήματος βάσης δεδομένων για τη διαχείρηση του πληροφοριακού συστήματος του νοσοκομείου "Υγειόπολης". Η βάση δεδομένων μας υλοποιήθηκε σε MySQL και καλύπτει όλες τις λειτουργίες που σχετίζονται με
- Ασθενείς
- Ιατρούς
- Νοσηλευτές
- Νοσηλείες
- Εξετάσεις
- Φαρμακευτική αγωγή
- Χειρουργικές επεμβάσεις
- Βάρδιες προσωπικού
- Αξιολογήσεις νοσηλείας
Το σύστημα σχεδιάστηκε με γνώμονα τις αρχές του σχεσιακού μοντέλου και περιλαμβάνει ER-EER διαγράμματα, περιορισμούς, triggers, και σύνθετα queries.

## Χαρακτηριστικά της βάσης

- Διαχείριση Ασθενών:
Αποθήκευση στοιχείων ασθενών, ασφαλιστικού φορέα και επαφών έκτακτης ανάγκης.
- Διαχείριση Προσωπικού:
Υποστήριξη ιατρών, νοσηλευτών και διοικητικού προσωπικού. Αποθήκευση στοιχείων, τμήματος και ρόλου του προσωπικού.
- Νοσηλείες και Διαγνώσεις:
Καταγραφή εισαγωγών, εξιτηρίων, διαγνώσεων ICD-10 και στοιχείων νοσηλείας.
- Φαρμακευτική Αγωγή:
Διαχείριση συνταγογραφήσεων, φαρμάκων, δραστικών ουσιών και αλλεργιών ασθενών.
- Ιατρικές Πράξεις και Χειρουργεία:
Καταγραφή επεμβάσεων, χειρουργείων, χειρουργικών αιθουσών και συμμετέχοντος προσωπικού.
- Βάρδιες Προσωπικού:
Οργάνωση βαρδιών και αναθέσεων προσωπικού.
- Αξιολογήσεις:
Υποστήριξη αξιολόγησης νοσηλείας και αξιολόγησης ιατρού.

## Παραδοχές

- Η ηλικία υπολογίζεται από την ημερομηνία γέννησης, και δεν αποθηκεύεται απευθείας.
- Η σειρά εξυπηρέτησης στο triage προκύπτει πάντα από το βαθμό προτεραιότητας και την ώρα άφιξης.
- Η ιεραρχία εποπτείας των γιατρών θεωρείται οτι έχει καταχωριστεί χωρίς κύκλους.
- Ο αριθμός δεδομένων της βάσης αυξήθηκε ώστε να επιστρέφουν ρεαλιστικά δεδομένα για όλα τα ερωτήματα.
- Η Ημερομηνία εξόδου νοσηλείας μπορεί να είναι κενή, εαν υπάρχει ενεργή-ανολοκλήρωτη νοσηλεία.
- Οι εικόνες των entities αποθηκεύονται σε έναν ενιαίο πίνακα, και όχι σε κάθε οντότητα ξεχωριστά. Έτσι αποθηκεύονται όλες οι απαραίτητες πληροφορίες για κάθε εικόνα.
- Το entity_image.entity_key αποθηκεύεται ως κείμενο ώστε να μπορεί να αναφέρεται σε διαφορετικά είδη κλειδιών.
- Τα queries που ζητούν συγκεκριμένο entity (γιατρό, ασθενή, ειδικότητα, ημερομηνία), υλοποιήθηκαν με σταθερές τιμές ώστε να επιστρέφουν αποτέλεσμα. Ωστόσο οι τιμές αυτές μπορούν να αλλάξουν εύκολα.
- Τα triggers εχουν τοποθετηθεί σε ξεχωριστό αρχείο, το οποίο εκτελείται μετά τη φόρτωση των αρχικών δεδομένων
- Για πρακτικούς λόγους το αρχείο φόρτωσης των δεδομένων έχει χωριστεί σε 5 επιμέρους αρχεία. 

## Τεχνολογίες που χρησιμοποιήθηκαν

- **MySQL:**
Χρησιμοποιήθηκε ως το βασικό σύστημα DBMS για τη δημιουργία και διαχείριση της βάσης hospitaldb
- **Draw.io:**
Χρησιμοποιήθηκε για τη σχεδίαση και την οπτική επεξεργασια του ER διαγράμματος της βάσης δεδομένων. Μέσω του εργαλείου οργανώθηκαν οι οντότητες, οι σχέσεις, τα cardinalities και η συνολική δομή του συστήματος.
- **MySQL Workbench:**
Χρησιμοποιήθηκε για τη σχεδίαση του relational diagram (EER), και την δημιουργία του τελικού relational schema, με τη χρήση forward engineer.
- **Python:**
Χρησιμοποιήθηκε για την παραγωγή του αρχείου load.sql . Έγινε ανάγνωση δεδομένων από αρχεία excel και η μετατροπή τους σε SQL INSERT statements.
- **OpenPyXL:**
Η βιβλιοθήκη αυτή χρησιμοποιήθηκε στη python για την ανάγνωση των excel αρχείων, την επεξεργασία των φύλλων εργασίας, τη μετατροπή ημερομηνιών και την εξαγωγή των δεδομένων προς SQL μορφή.
- **ChatGPT:**
Χρησιμοποιήθηκε υποστηρικτικά για τη δημιουργία ενδεικτικών δεδομένων και ονομάτων καθώς και των αντίστοιχων εικόνων. Επιπλέον χρησιμοποιήθηκε υποστηρικτικά για τη συγγραφή του Python script παραγωγής SQL αρχείων και την υλοποίηση βασικών τμημάτων της web εφαρμογής. Ο τελικός έλεγχος και η προσαρμογή του κώδικα έγιναν από την ομάδα.
- **Visual Studio Code:**
Χρησιμοποιήθηκε ως το βασικό περιβάλλον ανάπτυξης για τη συγγραφή των sql scripts, python scripts και README, καθώς και για τη γενικότερη οργάνωση του project.
- **MAMP:**
Χρησιμοποιήθηκε για τοπική εγκατάσταση και διαχείριση MySQL server σε περιβάλλον macOS.

## Εκτέλεση εφαρμογής σε MAMP

Η εφαρμογή βρίσκεται στον φάκελο code/hospital_app. 
Για να εκτελεστεί μέσω MAMP, ο φάκελος αυτός τοποθετείται μέσα στο htdocs.
Στη συνέχεια, αφού ξεκινήσουν Apache και MySQL, η εφαρμογή ανοίγει από τον browser μέσω localhost, ανάλογα με τη θέση του φακέλου μέσα στο htdocs.

## Ρυθμίσεις σύνδεσης με τη βάση
Οι ρυθμίσεις σύνδεσης βρίσκονται στο αρχείο:
code/hospital_app/config.php

Για MAMP χρησιμοποιούνται οι παρακάτω τιμές:
- DB_HOST: 127.0.0.1
- DB_PORT: 8889
- DB_NAME: hospitaldb
- DB_USER: root
- DB_PASS: root
Αν το MySQL port στο MAMP είναι διαφορετικό, αλλάζει μόνο η τιμή DB_PORT στο config.php.

## Φόρτωση της βάσης
Τα SQL αρχεία φορτώνονται από το phpMyAdmin με την παρακάτω σειρά:

1. sql/install/01_hospitaldb9_schema.sql
2. sql/load_parts/load10a.sql
3. sql/load_parts/load10b.sql
4. sql/load_parts/load10c.sql
5. sql/load_parts/load10d.sql
6. sql/load_parts/load10e.sql
7. sql/install/02_triggers_after_load.sql
Το πρώτο αρχείο δημιουργεί τη βάση και τους πίνακες.  
Τα αρχεία load10a.sql έως load10e.sql εισάγουν τα δεδομένα.  
Το αρχείο 02_triggers_after_load.sql προσθέτει τα triggers μετά τη φόρτωση των δεδομένων.

## Queries
Τα SQL queries της εργασίας βρίσκονται στον φάκελο:
sql/queries/
Τα ίδια queries έχουν ενσωματωθεί και στο Web UI μέσω του αρχείου:
